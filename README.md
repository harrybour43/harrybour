📸 Harry Bour — Portfólio Fotográfico & Processos Visuais
“A imagem não deve ser um monumento à eternidade, e sim o testemunho de uma efemeridade necessária.”

Este é o repositório oficial do portfólio digital de Harry Bour. Um espaço desenvolvido para exibir narrativas visuais, fotojornalismo e trabalhos autorais, focado no realismo hostil, na crueza do cotidiano de São Paulo e na exploração da luz e sombra (Low-key e Bodyscapes).

🌍 Site ao vivo: harrybour.com.br

🚀 Funcionalidades e UX/UI
O site utiliza uma arquitetura front-end limpa, focada em performance e estética minimalista:

Galeria Dinâmica (Masonry): Grid responsivo que preserva a proporção original das fotografias.

Lightbox Imersivo: Visualização em tela cheia com navegação por teclado (← → e Esc).

Acessibilidade Persistente (a11y): Painel com Alto Contraste, Daltonismo e Escala de Fonte. Preferências salvas via localStorage.

Proteção de Conteúdo: Bloqueio de clique direito, atalhos de inspeção e drag-and-drop de imagens.

Age Gate: Validação de idade para o Projeto Lumen (+18) via fricção intencional.

🗂 Estrutura do Site

⚙️ Tecnologias & Engenharia
Desenvolvimento Vanilla (sem frameworks), priorizando o controle total do código:

HTML5 Semântico: Foco em SEO e leitores de tela.

CSS3 Moderno: Variáveis (:root), Flexbox, CSS Grid e animações de transição suave.

JavaScript (ES6+): * Manipulação dinâmica de DOM e Fetch API.

Gerenciamento de estado e persistência de dados.

Interceptação de eventos de segurança.

Infraestrutura: Hospedagem via GitHub Pages com SSL/TLS ativo e domínio customizado.

🔍 SEO & Performance
Open Graph: Meta tags otimizadas para Rich Previews em redes sociais (WhatsApp/Instagram).

Sitemap XML: Estrutura de URLs limpas e hierarquia de prioridades.

Mixed Content Fix: Garantia de navegação 100% segura (HTTPS).

📁 Workflow de Atualização
A curadoria foi automatizada para não depender de edição no HTML:

Tratamento: Exportar imagem em JPG.

Nomenclatura: Usar padrão Nome-da-Foto-SP.jpg (será usado como legenda).

Upload: Salvar na pasta correspondente em assets/img/.

Publicação: Realizar o git commit e push. A galeria atualiza automaticamente.
